# 对话页面隔离功能 - 数据库迁移指南

## 功能说明

实现对话按页面隔离，让每个页面的对话历史互相独立：
- 沉降页面只显示沉降相关的对话
- 裂缝页面只显示裂缝相关的对话
- 温度页面只显示温度相关的对话
- 等等...

## 数据库迁移步骤

### 1. 登录 Supabase

访问：https://supabase.com/dashboard
登录你的项目

### 2. 打开 SQL Editor

在左侧菜单找到 "SQL Editor"

### 3. 执行迁移 SQL

复制并执行以下 SQL：

```sql
-- 为 assistant_conversations 表添加 page_path 字段以实现对话页面隔离
-- 执行时间: 2026-03-09

-- 1. 添加 page_path 字段
ALTER TABLE assistant_conversations
ADD COLUMN page_path TEXT;

-- 2. 为现有数据设置默认值（可选，如果有历史数据）
UPDATE assistant_conversations
SET page_path = '/settlement'
WHERE page_path IS NULL;

-- 3. 添加索引以提升查询性能
CREATE INDEX idx_assistant_conversations_page_path
ON assistant_conversations(page_path);

-- 4. 添加复合索引（page_path + updated_at）用于排序查询
CREATE INDEX idx_assistant_conversations_page_path_updated_at
ON assistant_conversations(page_path, updated_at DESC);

-- 5. 验证修改
SELECT column_name, data_type, is_nullable
FROM information_schema.columns
WHERE table_name = 'assistant_conversations'
ORDER BY ordinal_position;
```

### 4. 验证结果

执行成功后，你应该看到：
- `page_path` 字段已添加
- 两个索引已创建
- 表结构显示所有字段

## 代码修改说明

### 后端修改（已完成）

1. **db_service.py**
   - `create_conversation()` 添加 `page_path` 参数
   - `get_conversations()` 支持按 `page_path` 过滤
   - 返回数据包含 `pagePath` 字段

2. **api.py**
   - `GET /api/assistant/conversations` 支持 `page_path` 查询参数
   - `POST /api/assistant/conversations` 接收 `pagePath` 字段

### 前端修改（已完成）

1. **types.ts**
   - `Conversation` 接口添加 `pagePath?: string`

2. **api.ts**
   - `getConversations()` 支持 `pagePath` 参数
   - `createConversation()` 支持 `pagePath` 参数

3. **AssistantPanel.tsx**
   - 根据 `location.pathname` 加载对话
   - 创建对话时自动关联当前页面
   - 切换页面时重新加载对话列表

## 使用效果

### 迁移前
- 所有页面共享同一个对话列表
- 在沉降页面能看到裂缝页面的对话

### 迁移后
- 每个页面有独立的对话列表
- 沉降页面（`/settlement`）只显示沉降相关对话
- 裂缝页面（`/cracks`）只显示裂缝相关对话
- 温度页面（`/temperature`）只显示温度相关对话

## 注意事项

1. **历史数据处理**
   - 迁移脚本会将所有现有对话的 `page_path` 设置为 `/settlement`
   - 如果需要不同的默认值，修改 SQL 中的 `UPDATE` 语句

2. **性能优化**
   - 已添加索引，查询性能不会受影响
   - 复合索引支持按页面+时间排序

3. **兼容性**
   - `page_path` 字段允许为 NULL
   - 旧版本客户端仍可正常工作（会看到所有对话）

## 回滚方案

如果需要回滚：

```sql
-- 删除索引
DROP INDEX IF EXISTS idx_assistant_conversations_page_path_updated_at;
DROP INDEX IF EXISTS idx_assistant_conversations_page_path;

-- 删除字段
ALTER TABLE assistant_conversations
DROP COLUMN page_path;
```
