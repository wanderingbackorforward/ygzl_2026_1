# 悬浮小助手数据库初始化指南

## Supabase 数据库初始化步骤

### 1. 登录 Supabase 控制台

访问 [https://supabase.com/dashboard](https://supabase.com/dashboard)，选择你的项目。

### 2. 执行 SQL 脚本

1. 在左侧菜单中点击 **SQL Editor**
2. 点击 **New query** 创建新查询
3. 复制 `backend/modules/assistant/schema_supabase.sql` 文件的全部内容
4. 粘贴到 SQL 编辑器中
5. 点击 **Run** 按钮执行

### 3. 验证表创建成功

执行以下 SQL 查询验证：

```sql
-- 查看表结构
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
AND table_name IN ('assistant_conversations', 'assistant_messages');

-- 查看索引
SELECT indexname, tablename
FROM pg_indexes
WHERE schemaname = 'public'
AND tablename IN ('assistant_conversations', 'assistant_messages');
```

应该看到：
- `assistant_conversations` 表（对话表）
- `assistant_messages` 表（消息表）
- 6 个索引
- 1 个触发器

### 4. 测试插入数据

```sql
-- 插入测试对话
INSERT INTO assistant_conversations (id, title, role)
VALUES ('test-conv-001', '测试对话', 'researcher');

-- 插入测试消息
INSERT INTO assistant_messages (id, conversation_id, role, content, content_type)
VALUES
  ('test-msg-001', 'test-conv-001', 'user', '这是一条测试消息', 'text'),
  ('test-msg-002', 'test-conv-001', 'assistant', '这是AI的回复', 'markdown');

-- 查询验证
SELECT * FROM assistant_conversations;
SELECT * FROM assistant_messages;

-- 清理测试数据
DELETE FROM assistant_conversations WHERE id = 'test-conv-001';
```

### 5. 配置 Row Level Security (RLS) - 可选

如果需要启用 RLS（推荐生产环境）：

```sql
-- 启用 RLS
ALTER TABLE assistant_conversations ENABLE ROW LEVEL SECURITY;
ALTER TABLE assistant_messages ENABLE ROW LEVEL SECURITY;

-- 创建策略（允许所有操作，后续可根据需求调整）
CREATE POLICY "允许所有操作 assistant_conversations"
ON assistant_conversations
FOR ALL
USING (true)
WITH CHECK (true);

CREATE POLICY "允许所有操作 assistant_messages"
ON assistant_messages
FOR ALL
USING (true)
WITH CHECK (true);
```

## 表结构说明

### assistant_conversations（对话表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(50) | 对话唯一标识（主键） |
| title | VARCHAR(200) | 对话标题 |
| role | VARCHAR(20) | 角色类型（researcher/worker/reporter） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间（自动更新） |
| deleted_at | TIMESTAMP | 软删除时间 |

### assistant_messages（消息表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(50) | 消息唯一标识（主键） |
| conversation_id | VARCHAR(50) | 所属对话ID（外键） |
| role | VARCHAR(20) | 消息角色（user/assistant） |
| content | TEXT | 消息内容 |
| content_type | VARCHAR(20) | 内容类型（text/markdown/chart/table） |
| metadata | JSONB | 元数据（JSON格式） |
| created_at | TIMESTAMP | 创建时间 |

## 常见问题

### Q1: 执行 SQL 时报错 "relation already exists"

**原因**：表已经存在

**解决**：
- 如果是测试环境，可以先删除表：
  ```sql
  DROP TABLE IF EXISTS assistant_messages CASCADE;
  DROP TABLE IF EXISTS assistant_conversations CASCADE;
  ```
- 然后重新执行 `schema_supabase.sql`

### Q2: 外键约束错误

**原因**：尝试插入消息时对话不存在

**解决**：确保先创建对话，再插入消息

### Q3: 时区问题

**说明**：PostgreSQL 使用 `TIMESTAMP WITH TIME ZONE`，会自动处理时区转换
