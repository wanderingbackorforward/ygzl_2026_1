# 数据总览 V2 天才重构计划

> 设计哲学：一个界面，三种角色（工程师/科研/管理），零学习成本
> 核心理念：把后端90%被浪费的能力全部释放到前端

## 现有问题（PUA自检）

- 只用了2个API，后端8大能力模块全部浪费
- 力导向图花哨但信息密度为零
- 工程人员没有施工指导参数
- 科研人员看不到任何算法深度
- 没有时间维度，没有趋势对比

## 实施步骤（分6步，每步完成打勾）

### 第1步：左侧指标脉搏栏 ⬜
- 5个紧凑指标卡（安全评分/沉降/裂缝/温度/盾构）
- 调用API：`/summary` + `/analysis/v2/settlement` + `/analysis/v2/temperature`
- 每个卡片：核心数值 + 迷你sparkline + 状态色块
- 220px固定宽度，垂直排列

### 第2步：诊断引擎V2升级 ⬜
- 接入温度分析数据（当前只有沉降）
- 接入施工事件数据（`/advanced/events`）
- 接入沉降-裂缝联合分析（`/advanced/joint/summary`）
- 综合多源数据计算健康分（不再只看沉降）
- 生成更丰富的行动指令（含具体参数）

### 第3步：核心画布 — 工程师视图 ⬜
- 地质剖面图（`/advanced/profile/layers` + `/advanced/profile/data`）
- 沉降曲线叠加在地层上
- 施工事件标注（`/advanced/events/timeline`）
- 点击事件可查看影响分析（`/advanced/events/{id}/impact`）

### 第4步：核心画布 — 科研视图 ⬜
- 多算法预测对比（展示后端ML模型结果）
- 空间相关性热力图（利用spatial_correlation）
- 沉降-裂缝相关性散点图（`/advanced/joint/correlation`）
- 关键统计指标展示（R², Moran's I, 相关系数）

### 第5步：核心画布 — 管理视图 ⬜
- 风险矩阵（概率×影响 散点图）
- 趋势概览（多点沉降趋势合并）
- 异常占比饼图 + 建议优先级排序
- 30天预测汇总

### 第6步：底栏 + 细节打磨 ⬜
- 施工事件滚动时间线
- 自动刷新优化
- 空状态/加载状态/错误状态处理
- 移动端适配检查

## 数据来源映射

| 组件 | API端点 | 数据表 |
|------|---------|--------|
| 安全评分卡 | /summary | processed_settlement_data |
| 沉降概况卡 | /analysis/v2/settlement | processed_settlement_data |
| 裂缝概况卡 | /advanced/joint/summary | crack_monitoring_data |
| 温度概况卡 | /analysis/v2/temperature | temperature相关表 |
| 盾构状态卡 | 新接口或直读 | tbm_telemetry |
| 地质剖面 | /advanced/profile/* | tunnel_profile_config + geological_layers |
| 施工事件 | /advanced/events/* | construction_events |
| 联合分析 | /advanced/joint/* | settlement_crack_mapping |
| ML预测 | /api/ml/* | ml_models模块 |
| 空间分析 | spatial_correlation | processed_settlement_data |

## 技术约束
- 纯前端改造，不新增后端文件（除非必要的API端点缺失）
- 全中文界面
- 深色主题 text-white
- Flexbox布局规范
- ECharts图表
