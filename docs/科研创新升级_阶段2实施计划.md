## 二、技术方案

### 2.1 知识图谱Schema设计

#### 实体类型 (Nodes)

1. **MonitoringPoint** (监测点)
   - 属性: point_id, name, type, location, elevation, install_date
   - 数量: 约100-200个

2. **Sensor** (传感器)
   - 属性: sensor_id, type, model, accuracy, install_date
   - 数量: 约300-500个

3. **ConstructionEvent** (施工事件)
   - 属性: event_id, name, type, start_date, end_date, location, impact_level
   - 数量: 约50-100个

4. **GeologicalStructure** (地质结构)
   - 属性: structure_id, name, type, depth, material
   - 数量: 约20-50个

5. **WeatherCondition** (气象条件)
   - 属性: date, temperature, rainfall, humidity, wind_speed
   - 数量: 约365个(每天一条)

6. **Anomaly** (异常事件)
   - 属性: anomaly_id, date, type, severity, point_id
   - 数量: 约100-200个

7. **MaintenanceRecord** (维护记录)
   - 属性: record_id, date, type, description, cost
   - 数量: 约50-100个

#### 关系类型 (Relationships)

1. **LOCATED_AT** (位于)
   - 监测点 -> 地质结构
   - 属性: distance

2. **HAS_SENSOR** (拥有传感器)
   - 监测点 -> 传感器
   - 属性: install_date

3. **NEAR** (邻近)
   - 监测点 -> 监测点
   - 属性: distance, direction

4. **AFFECTED_BY** (受影响)
   - 监测点 -> 施工事件
   - 属性: impact_score, time_lag

5. **CAUSED_BY** (由...引起)
   - 异常 -> 施工事件/气象条件
   - 属性: confidence, evidence

6. **DETECTED_AT** (检测到)
   - 异常 -> 监测点
   - 属性: date, severity

7. **MAINTAINED_BY** (维护)
   - 监测点 -> 维护记录
   - 属性: date

8. **CORRELATED_WITH** (相关)
   - 监测点 -> 监测点
   - 属性: correlation_coefficient

   - 基于时间戳计算先后

   - 置信度阈值: 0.7

## 四、关键代码示例

### 4.1 构建图谱脚本

```python
# backend/modules/knowledge_graph/scripts/build_graph.py

from knowledge_graph import KnowledgeGraphBuilder
import requests
import os

def main():
    # 1. 连接Neo4j
    builder = KnowledgeGraphBuilder(
        uri="bolt://localhost:7687",
        user="neo4j",
        password="password"
    )

    # 2. 清空旧数据(可选)
    # builder.clear_graph()

    # 3. 创建索引
    builder.create_indexes()

    # 4. 抽取数据
    points_data = fetch_monitoring_points()
    sensors_data = fetch_sensors()
    events_data = fetch_construction_events()

    # 5. 创建实体
    builder.create_monitoring_points(points_data)
    builder.create_sensors(sensors_data)
    builder.create_construction_events(events_data)

    # 6. 创建关系
    builder.create_spatial_relationships(points_data)
    builder.create_causal_relationships(events_data, points_data)

    # 7. 验证
    stats = builder.get_graph_stats()
    print(f"实体数量: {stats['nodes']}")
    print(f"关系数量: {stats['relationships']}")

    builder.close()

if __name__ == '__main__':
    main()
```

### 4.2 查询示例

```python
# 查询1: 找到所有邻近S1的监测点
query = """
MATCH (p1:MonitoringPoint {point_id: 'S1'})-[r:NEAR]->(p2:MonitoringPoint)
RETURN p2.point_id, p2.name, r.distance
ORDER BY r.distance
"""

# 查询2: 找到受施工影响的监测点
query = """
MATCH (p:MonitoringPoint)-[r:AFFECTED_BY]->(e:ConstructionEvent)
WHERE r.impact_score > 0.7
RETURN p.point_id, e.name, r.impact_score
ORDER BY r.impact_score DESC
"""

# 查询3: 找到异常的因果链
query = """
MATCH path = (a:Anomaly)-[:CAUSED_BY*1..3]->(cause)
WHERE a.severity = 'high'
RETURN path
LIMIT 10
"""
```

## 六、风险与应对

### 风险1: Neo4j安装困难
**影响**: 无法搭建图数据库

**应对**:
- 使用Docker简化安装
- 使用Neo4j Aura云服务
- 使用其他图数据库(如ArangoDB)

### 风险2: 数据量不足
**影响**: 图谱实体和关系数量达不到目标

**应对**:
- 生成模拟数据
- 扩展实体类型(如添加维护记录、报警记录)
- 增加历史数据

### 风险3: 关系抽取不准确
**影响**: 因果关系等复杂关系不可靠

**应对**:
- 使用规则+模型混合方法
- 人工标注部分数据
- 降低置信度阈值
