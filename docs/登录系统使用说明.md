# 登录系统使用说明

## 概述

本系统集成了一个松耦合的登录认证模块，可以通过环境变量轻松启用或禁用。

## 特性

- **松耦合设计**：认证模块独立，不侵入现有代码
- **环境变量控制**：通过 `VITE_AUTH_ENABLED` 控制是否启用
- **默认禁用**：开箱即用，无需登录
- **一键跳过**：即使启用登录，也可以点击"跳过登录"按钮
- **持久化登录**：登录状态保存在 localStorage
- **简单密码验证**：用户名+密码验证（可扩展为后端验证）

## 启用/禁用登录

### 方法 1：环境变量（推荐）

在 `frontend/.env` 文件中添加：

```bash
# 启用登录
VITE_AUTH_ENABLED=true

# 禁用登录（默认）
VITE_AUTH_ENABLED=false
```

### 方法 2：直接修改代码

编辑 `frontend/src/contexts/AuthContext.tsx`，修改第 18 行：

```typescript
// 启用登录
const IS_AUTH_ENABLED = true;

// 禁用登录（默认）
const IS_AUTH_ENABLED = false;
```

## 使用方式

### 1. 正常登录

1. 输入用户名和密码
2. 点击"登录"按钮
3. 登录成功后进入系统

### 2. 跳过登录

1. 点击"跳过登录"按钮
2. 以访客身份进入系统
3. 跳过状态会保存，下次访问自动跳过

### 3. 退出登录

1. 点击右上角用户名
2. 在下拉菜单中点击"退出登录"
3. 返回登录页面

## 扩展为后端验证

当前使用前端硬编码的用户数据，生产环境应改为后端验证：

### 修改 `AuthContext.tsx` 的 `login` 函数：

```typescript
const login = useCallback(async (username: string, password: string) => {
  try {
    // 调用后端 API
    const response = await fetch('/api/auth/login', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ username, password }),
    });

    const result = await response.json();

    if (result.success) {
      const userData: User = {
        username: result.data.username,
        displayName: result.data.displayName,
        role: result.data.role,
      };

      setUser(userData);
      setIsAuthenticated(true);
      localStorage.setItem(AUTH_STORAGE_KEY, JSON.stringify(userData));
      localStorage.removeItem(AUTH_SKIP_KEY);

      return { success: true };
    } else {
      return { success: false, message: result.message };
    }
  } catch (error) {
    return { success: false, message: '网络错误' };
  }
}, []);
```

### 后端 API 示例（Flask）：

```python
@app.route('/api/auth/login', methods=['POST'])
def login():
    data = request.get_json()
    username = data.get('username')
    password = data.get('password')

    # 验证用户名和密码（从数据库查询）
    user = User.query.filter_by(username=username).first()

    if user and user.check_password(password):
        return jsonify({
            'success': True,
            'data': {
                'username': user.username,
                'displayName': user.display_name,
                'role': user.role
            }
        })
    else:
        return jsonify({
            'success': False,
            'message': '用户名或密码错误'
        }), 401
```

## 回滚方案

如果登录系统出现问题，可以快速回滚：

### 方法 1：禁用登录（推荐）

设置环境变量 `VITE_AUTH_ENABLED=false`，重新构建前端。

### 方法 2：移除认证模块

1. 编辑 `frontend/src/main.tsx`
2. 移除以下代码：

```typescript
// 移除这些导入
import { AuthProvider } from './contexts/AuthContext'
import { AuthGuard } from './components/auth/AuthGuard'

// 移除 AuthProvider 和 AuthGuard 包裹
function App() {
  return (
    // <AuthProvider>  // 删除
    //   <AuthGuard>   // 删除
        <ModulesProvider>
          {/* ... 其他代码 ... */}
        </ModulesProvider>
    //   </AuthGuard>  // 删除
    // </AuthProvider> // 删除
  )
}
```

3. 重新构建前端

## 常见问题

### Q: 忘记密码怎么办？

A: 当前版本使用硬编码账号，无法找回密码。可以使用测试账号或清除 localStorage：

```javascript
// 在浏览器控制台执行
localStorage.removeItem('auth_user');
localStorage.removeItem('auth_skip');
location.reload();
```

### Q: 如何添加新用户？

A: 编辑 `frontend/src/contexts/AuthContext.tsx`，在 `USERS` 数组中添加：

```typescript
const USERS = [
  // ... 现有用户
  { username: 'newuser', password: 'password123', displayName: '新用户', role: 'user' },
];
```

### Q: 登录状态保存多久？

A: 登录状态保存在 localStorage，除非手动退出或清除浏览器数据，否则永久有效。

### Q: 如何实现自动登出？

A: 可以添加 Token 过期机制，在 `AuthContext.tsx` 中实现：

```typescript
// 保存登录时间
localStorage.setItem('auth_timestamp', Date.now().toString());

// 检查是否过期（例如 24 小时）
const timestamp = localStorage.getItem('auth_timestamp');
if (timestamp && Date.now() - parseInt(timestamp) > 24 * 60 * 60 * 1000) {
  logout();
}
```

## 安全建议

1. **生产环境必须使用后端验证**：前端验证不安全
2. **使用 HTTPS**：防止密码被窃听
3. **密码加密传输**：使用 bcrypt 或类似算法
4. **添加验证码**：防止暴力破解
5. **限制登录尝试次数**：防止暴力攻击
6. **使用 JWT Token**：替代 localStorage 存储用户信息
7. **定期更新密码**：强制用户定期修改密码
