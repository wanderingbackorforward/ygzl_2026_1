# 算法升级实施总结 - 第三阶段：关联分析

## 实施时间
2026-03-05

## 实施目标
在高级分析模块中实现关联分析功能，包括因果分析和空间关联分析，帮助用户理解施工事件对沉降的影响以及监测点位之间的空间关联关系。

## 实施内容

### 1. 新增组件

#### 1.1 EventImpactChart.tsx
**功能**：事件影响分析可视化组件
- 使用 ECharts 绘制处理组与对照组的对比曲线
- 显示事件发生时间的标记线
- 展示 DID/SCM 分析结果（处理效应、置信区间）
- 支持反事实曲线显示（SCM方法）

**关键特性**：
- 双Y轴设计，清晰对比处理组和对照组
- 事件前后期数据分段显示
- 置信区间可视化
- 详细的统计结果卡片

#### 1.2 SpatialCorrelationHeatmap.tsx
**功能**：空间相关性热力图组件
- 使用 ECharts 绘制相关系数矩阵热力图
- 显示空间聚类信息
- 颜色映射：-1（强负相关）到 +1（强正相关）

**关键特性**：
- 11色渐变色阶（蓝-黄-红）
- 聚类卡片展示（点位分组）
- 相关性图例说明
- 交互式 tooltip 显示详细相关系数

#### 1.3 CausalAnalysis.tsx
**功能**：因果分析配置面板
- 处理组点位选择
- 事件日期和名称输入
- 对照组点位多选
- 分析方法选择（DID/SCM）
- 时间窗口配置

**关键特性**：
- 完整的表单验证
- 自动对照组选择（未手动选择时）
- 加载状态和错误处理
- 集成 EventImpactChart 展示结果

#### 1.4 CorrelationDashboard.tsx
**功能**：关联分析主面板
- 统计卡片（监测点位、空间聚类、分析方法、状态）
- 分析类型切换（因果分析/空间关联）
- 距离阈值配置（空间关联）
- 集成 CausalAnalysis 和 SpatialCorrelationHeatmap

**关键特性**：
- 双模式切换（因果/空间）
- 独立的加载和错误状态管理
- 刷新按钮支持重新加载数据
- 响应式布局

### 2. 页面集成

#### 2.1 AdvancedAnalysis.tsx 修改
- 添加 `'correlation'` 到 `TabType` 类型
- 新增"关联分析" Tab 按钮（图标：project-diagram）
- 创建 `CorrelationTab` 组件
- 传递 pointIds 数据到 CorrelationDashboard

### 3. API 集成

#### 3.1 因果分析 API
```
POST /api/ml/causal/event-impact
Body: {
  point_id: string,
  event_date: string,
  control_point_ids?: string[],
  method: 'DID' | 'SCM',
  window_days: number
}
```

#### 3.2 空间关联 API
```
GET /api/ml/spatial/correlation?distance_threshold=50
Response: {
  success: boolean,
  points: Array<{point_id, x, y}>,
  correlation_matrix: number[][],
  adjacency_matrix: number[][],
  clusters: number[][],
  cluster_count: number
}
```
