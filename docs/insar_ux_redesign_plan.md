# InSAR 模块交互设计重构计划

## 问题诊断

当前 InSAR 页面存在严重的交互混乱问题：

### 核心问题
1. **功能入口混乱** — 用户不知道能干什么
   - 顶部有 4 组控件（地图/预警、原生/外链、数据集、指标）平铺，无层次
   - "原生模式"/"外链模式"对普通用户无意义
   - 参数控件（eps、minPts、里程分箱）暴露在顶部，专业性过强

2. **信息架构缺失** — 没有清晰的任务流
   - 用户不知道第一步该做什么
   - 地图和预警/建议是平级 tab，但实际是"查看→分析→决策"的流程
   - 危险区、时序图、施工建议散落各处

3. **视觉层次混乱** — 所有元素同等重要
   - 12 个按钮/输入框挤在顶部
   - 关键操作（查看高风险点）埋在页面中部
   - 字体大小、颜色、间距无规律

## 设计原则（乔布斯式）

### 1. 零学习成本
- 用户打开页面就知道能干什么
- 主要操作不超过 3 个
- 高级功能折叠隐藏

### 2. 任务导向
- 按照"监测→预警→决策"的工作流组织
- 每个阶段有明确的视觉分区
- 用户不需要思考"下一步做什么"

### 3. 渐进式披露
- 默认显示：地图 + 风险概览（3 个数字）
- 点击风险点 → 展开详情面板
- 高级参数（阈值、聚类参数）放在设置抽屉

### 4. 信息密度优化
- 用徽章替代大卡片（危险 3 / 预警 12 / 正常 156）
- 用侧边面板替代 tab 切换
- 图表紧凑排列，移除冗余说明文字

### 关键改进点

#### 1. 顶栏简化（从 12 个控件 → 3 个）
- **保留**：数据集下拉、设置按钮
- **移除**：地图/预警 tab（改为侧边面板）、原生/外链切换（外链改为设置项）、指标切换（移到设置）、所有参数输入框（移到设置）

#### 2. 默认视图（地图 + 风险概览）
- 地图占 70% 宽度，右侧 30% 显示风险概览
- 风险概览只显示 3 个数字 + 高风险点列表（Top 5）
- 点击任意高风险点 → 地图定位 + 右侧展开详情面板

#### 3. 详情面板（侧边滑出）
- **触发**：点击高风险点、点击地图上的点
- **内容**：点位信息、时序图、施工建议、AI 生成建议
- **操作**：关闭按钮、上一个/下一个点位切换

#### 4. 设置抽屉（侧边滑出）
- **触发**：点击顶栏"⚙️ 设置"按钮
- **内容**：显示模式（速度/关键日期/阈值分级）、阈值设置、高级选项（视窗裁剪、危险区、聚类参数）
- **操作**：关闭按钮、重置按钮

#### 5. 移除"预警/建议" tab
- 风险统计图表移到详情面板底部（折叠）
- 施工建议清单直接显示在详情面板
- 全局建议改为浮动按钮（右下角）

## 预期效果

### 改进前
- 用户打开页面：12 个控件，不知道从哪开始
- 查看高风险点：需要切换 tab、滚动页面、点击按钮
- 修改参数：在顶部找输入框，不知道改了会影响什么

### 改进后
- 用户打开页面：地图 + 3 个数字（危险 3），立即知道有问题
- 查看高风险点：点击列表 → 地图定位 + 详情面板滑出，一步到位
- 修改参数：点击设置按钮 → 抽屉滑出，所有参数分组清晰

## 风险与注意事项

1. **不要一次性重写整个文件** — 按步骤拆分，每步完成后测试
2. **保留现有功能** — 只改交互，不删功能（除非明确冗余）
3. **渐进式迁移** — 先实现新布局，再逐步迁移功能
4. **测试关键路径** — 每步完成后测试：选点 → 查看详情 → 修改参数 → 更新地图
