## 二、技术实现详情

### 1. 知识图谱Schema设计

**文件**: `docs/知识图谱Schema设计.md`

**实体类型** (6种):
1. **MonitoringPoint** (监测点) - 沉降、温度、裂缝、振动监测点
2. **Sensor** (传感器) - 型号、精度、校准日期
3. **ConstructionEvent** (施工事件) - 开挖、爆破、注浆、支护
4. **GeologicalStructure** (地质结构) - 断层、地层、含水层、岩溶
5. **WeatherCondition** (气象条件) - 温度、湿度、降雨、风速
6. **Anomaly** (异常事件) - 突变、加速、波动、趋势异常

**关系类型** (8种):
1. **SPATIAL_NEAR** (空间邻近) - 距离、方向
2. **TEMPORAL_BEFORE** (时间先后) - 时间差
3. **CAUSES** (因果关系) - 置信度、效应大小、滞后天数
4. **CORRELATES_WITH** (相关性) - 相关系数、显著性
5. **HAS_SENSOR** (拥有传感器)
6. **LOCATED_IN** (位于)
7. **AFFECTED_BY** (受影响) - 影响程度、时间范围
8. **DETECTED_AT** (检测到)

**索引设计**:
- 节点索引: point_id, sensor_id, event_id, date等
- 关系索引: 自动优化

**典型查询示例**:
```cypher
// 查找邻近点
MATCH (p:MonitoringPoint {point_id: 'S1'})-[r:SPATIAL_NEAR]-(neighbor)
WHERE r.distance < 50
RETURN neighbor

// 查找因果链
MATCH (e:ConstructionEvent)-[c:CAUSES]->(a:Anomaly)-[:DETECTED_AT]->(p)
WHERE a.severity IN ['critical', 'high']
RETURN e, a, p, c

// 查找传播路径
MATCH path = (p1)-[:SPATIAL_NEAR*1..3]-(p2)
WHERE EXISTS((p1)<-[:DETECTED_AT]-(:Anomaly))
RETURN path
```

### 3. 因果推理引擎

**文件**: `backend/modules/ml_models/causal_reasoning.py`

**核心功能**:

#### 3.1 因果发现 (Causal Discovery)
```python
engine = CausalReasoningEngine()

# Granger因果检验
causal_pairs = engine.discover_causal_relationships(
    data, method='granger', max_lag=5
)

# 相关性分析（初步筛选）
causal_pairs = engine.discover_causal_relationships(
    data, method='correlation'
)

# PC算法（基于条件独立性）
causal_pairs = engine.discover_causal_relationships(
    data, method='pc'
)
```

**Granger因果检验原理**:
- 如果X的历史信息能显著改善Y的预测，则X Granger因果Y
- 使用F检验判断显著性
- 返回最优滞后阶数和置信度

#### 3.2 因果效应估计 (Causal Effect Estimation)
```python
# 双重差分法 (DID)
result = engine.estimate_causal_effect(
    data, treatment_var='施工', outcome_var='沉降', method='DID'
)

# 合成控制法 (SCM)
result = engine.estimate_causal_effect(
    data, treatment_var='施工', outcome_var='沉降',
    method='SCM', control_vars=['S2', 'S3', 'S4']
)

# 倾向得分匹配 (PSM)
result = engine.estimate_causal_effect(
    data, treatment_var='施工', outcome_var='沉降',
    method='PSM', control_vars=['温度', '湿度']
)

# 工具变量法 (IV)
result = engine.estimate_causal_effect(
    data, treatment_var='施工', outcome_var='沉降',
    method='IV', control_vars=['降雨量']
)
```

**方法对比**:
| 方法 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| DID | 有处理组和对照组 | 简单直观 | 需要平行趋势假设 |
| SCM | 单个处理单位 | 构造反事实 | 需要多个对照单位 |
| PSM | 观察性数据 | 平衡协变量 | 需要无混淆假设 |
| IV | 存在内生性 | 解决内生性 | 需要有效工具变量 |

#### 3.3 反事实推理 (Counterfactual Reasoning)
```python
# "如果不发生施工，沉降会怎样？"
result = engine.counterfactual_inference(
    data,
    treatment_var='施工',
    outcome_var='沉降',
    intervention={'var': '施工', 'value': 0}
)

# 返回: 实际结果 vs 反事实结果
# {
#     'actual_mean': -25.5,
#     'counterfactual_mean': -15.2,
#     'effect': -10.3,
#     'effect_percentage': -40.4
# }
```

#### 3.4 干预效应估计 (Intervention Effect)
```python
# "采取加固措施，沉降会减少多少？"
result = engine.estimate_intervention_effect(
    data,
    intervention={'var': '加固强度', 'change': +10},
    outcome_var='沉降'
)

# 返回: 基线 vs 干预后
# {
#     'baseline_mean': -25.5,
#     'intervention_mean': -18.3,
#     'effect': +7.2,
#     'effect_percentage': +28.2
# }
```

**技术特点**:
- 支持5种因果推断方法
- 自动选择最优滞后阶数
- 提供置信度和效应大小
- 支持反事实推理和干预分析

### 4. KGQA问答系统

**文件**: `backend/modules/ml_models/kgqa.py`

**核心功能**:

#### 4.1 意图识别
支持6种问题类型:
1. **entity_query** (实体查询) - "S1的状态是什么？"
2. **neighbor_query** (邻近查询) - "S1附近有哪些监测点？"
3. **causal_query** (因果查询) - "哪些施工事件导致了异常？"
4. **count_query** (统计查询) - "有多少个高风险监测点？"
5. **path_query** (路径查询) - "S1到S5的传播路径是什么？"
6. **risk_query** (风险查询) - "高风险监测点有哪些？"

#### 4.2 实体识别
自动提取:
- 监测点ID: S1, S2, T1, C1等
- 事件ID: EVENT_001, EVENT_002等
- 异常ID: ANOMALY_001等

#### 4.3 Cypher生成
```python
kgqa = KGQA(kg_builder)

# 问题: "S1附近50米内有哪些监测点？"
# 生成Cypher:
"""
MATCH (p:MonitoringPoint {point_id: 'S1'})-[r:SPATIAL_NEAR]-(neighbor)
WHERE r.distance < 50
RETURN neighbor.point_id, neighbor.name, r.distance, r.direction
ORDER BY r.distance
"""

# 问题: "哪些施工事件导致了高严重度异常？"
# 生成Cypher:
"""
MATCH (e:ConstructionEvent)-[c:CAUSES]->(a:Anomaly)-[:DETECTED_AT]->(p)
WHERE a.severity IN ['critical', 'high']
RETURN e.name, a.date, p.point_id, c.confidence, c.effect_size
ORDER BY c.confidence DESC
"""
```

#### 4.4 答案生成
```python
result = kgqa.answer("S1附近有哪些监测点？")

# 返回:
# {
#     'success': True,
#     'question': 'S1附近有哪些监测点？',
#     'intent': 'neighbor_query',
#     'entities': ['S1'],
#     'cypher': '...',
#     'records': [...],
#     'answer': '找到 5 个邻近监测点:\n1. S2 (沉降监测点S2) - 距离 25.3米，方向 NE\n...'
# }
```

**技术特点**:
- 基于正则表达式的意图识别
- 自动提取实体
- 模板化Cypher生成
- 自然语言答案生成
